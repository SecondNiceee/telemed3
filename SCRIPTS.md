Если пользователь с указанным email уже существует, обновляет его роль на `admin`.

**Учетные данные (хардкод в скрипте):**
- Email: `col1596321@gmail.com`
- Пароль: `11559966332211kkKK`

**Запуск:**
```bash
pnpm tsx scripts/create-admin.ts
```

**Требования:**
- Должен быть настроен `.env` файл с переменными `DATABASE_URL` и `PAYLOAD_SECRET`

## Создание врачей

**Файл:** `scripts/create-doctors.ts`

**Описание:**  
Создает тестовых врачей в базе данных. Автоматически создает организацию (если не существует) и привязывает врачей к категориям.  
**Важно:** При запуске скрипта врачи автоматически получают расписание на 7 дней вперед с 3 консультациями в день (09:00, 12:00, 15:00).  
Если врач с таким email уже существует, скрипт обновляет его данные и расписание.

**Врачи (определены в `scripts/seed-data.config.ts`):**
- 2 кардиолога
- 2 невролога  
- 2 терапевта

**Запуск:**
```bash
# Сначала создайте организацию
pnpm tsx scripts/create-organisation.ts

# Затем создайте категории
pnpm tsx scripts/create-categories.ts

# Затем создайте врачей с расписанием
pnpm tsx scripts/create-doctors.ts
```

**Требования:**
- Должен быть настроен `.env` файл
- Организация должна быть создана
- Категории должны быть созданы (скрипт `create-categories.ts`)

**Учетные данные врачей:**
- Пароль для всех врачей: `Doctor123!`
- Email врачей указаны в конфиге (например: `ivanov.cardio@telemed.local`)

**Расписание:**
- Автоматически создается на 7 дней вперед от текущей даты
- 3 консультации в день: 09:00, 12:00, 15:00
- Слот длительности определяется параметром `slotDuration` в конфиге

## Создание пользователей

**Файл:** `scripts/create-users.ts`

**Описание:**  
Создает 10 тестовых пользователей (пациентов) в базе данных. Все пользователи создаются с верифицированным email (`_verified: true`), как в скрипте администратора.  
Если пользователь с таким email уже существует, скрипт обновляет его данные.

**Пользователи (определены в `scripts/seed-data.config.ts`):**
- 10 русских имен и фамилий
- Email вида: `volkov@test.local`, `sokolova@test.local` и т.д.
- Все с паролем: `User123!`

**Запуск:**
```bash
pnpm tsx scripts/create-users.ts
```

**Требования:**
- Должен быть настроен `.env` файл с переменными `DATABASE_URL` и `PAYLOAD_SECRET`
- База данных должна быть доступна

**Пример вывода:**
```
🚀 Подключение к Payload CMS...
✅ Подключение установлено
✅ Пользователь создан: Александр Волков (ID: user123)
✅ Пользователь создан: Мария Соколова (ID: user124)

✅ Всего создано/обновлено: 10 пользователей
🎉 Готово!
```

## Конфигурация данных

**Файл:** `scripts/seed-data.config.ts`

**Описание:**  
Централизованный конфиг с данными для заполнения базы. Содержит:
- `USERS` - массив пользователей (пациентов)
- `CATEGORIES` - массив категорий врачей
- `DOCTORS` - массив врачей с полной информацией
- `DEFAULT_ORGANISATION` - данные организации по умолчанию

Редактируйте этот файл, чтобы изменить тестовые данные.

## Порядок запуска скриптов

**Рекомендуемый порядок для инициализации проекта:**

```bash
# 1. Создание администратора (доступ в админ-панель)
pnpm tsx scripts/create-admin.ts

# 2. Создание организации
pnpm tsx scripts/create-organisation.ts

# 3. Создание категорий врачей
pnpm tsx scripts/create-categories.ts

# 4. Создание врачей с расписанием
pnpm tsx scripts/create-doctors.ts

# 5. Создание тестовых пользователей
pnpm tsx scripts/create-users.ts
```

После выполнения всех скриптов в системе будут:
- ✅ Администратор для управления CMS
- ✅ Организация здравоохранения
- ✅ Категории врачей (специализации)
- ✅ 6 врачей с расписанием на неделю
- ✅ 10 тестовых пользователей для записи на консультации
