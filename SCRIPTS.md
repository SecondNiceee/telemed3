## Создание категорий врачей

**Файл:** `scripts/create-categories.ts`

**Описание:**  
Создает категории (специализации) врачей в базе данных. Если категория с таким slug уже существует, пропускает её.

**Категории (определены в `scripts/seed-data.config.ts`):**
- Кардиология (`cardiology`)
- Неврология (`neurology`)
- Терапия (`therapy`)

**Запуск:**
```bash
pnpm tsx scripts/create-categories.ts
```

**Требования:**
- Должен быть настроен `.env` файл с переменными `DATABASE_URL` и `PAYLOAD_SECRET`

## Создание врачей

**Файл:** `scripts/create-doctors.ts`

**Описание:**  
Создает тестовых врачей в базе данных. Автоматически создает организацию (если не существует) и привязывает врачей к категориям.

**Врачи (определены в `scripts/seed-data.config.ts`):**
- 2 кардиолога
- 2 невролога  
- 2 терапевта

**Запуск:**
```bash
# Сначала создайте категории!
pnpm tsx scripts/create-categories.ts

# Затем создайте врачей
pnpm tsx scripts/create-doctors.ts
```

**Требования:**
- Должен быть настроен `.env` файл
- Категории должны быть созданы (скрипт `create-categories.ts`)

**Учетные данные врачей:**
- Пароль для всех врачей: `Doctor123!`
- Email врачей указаны в конфиге (например: `ivanov.cardio@telemed.local`)

**Организация по умолчанию:**

- Email: `org@telemed.local`

## Конфигурация данных

**Файл:** `scripts/seed-data.config.ts`

**Описание:**  
Централизованный конфиг с данными для заполнения базы. Содержит:
- `CATEGORIES` - массив категорий врачей
- `DOCTORS` - массив врачей с полной информацией
- `DEFAULT_ORGANISATION` - данные организации по умолчанию

Редактируйте этот файл, чтобы изменить тестовые данные.
