# Логика чата и консультации

## Обзор

Этот документ описывает полную логику работы чата и консультаций в телемедицинской платформе, включая управление статусами консультации и блокировку чата через WebSocket.

---

## Статусы консультации (Appointment)

| Статус | Описание |
|--------|----------|
| `confirmed` | Консультация подтверждена, ожидает начала |
| `in_progress` | Консультация активна (врач начал) |
| `completed` | Консультация завершена |
| `cancelled` | Консультация отменена |

---

## Поле chatBlocked

В коллекции `Appointments` есть поле `chatBlocked`:

```typescript
{
  name: 'chatBlocked',
  type: 'checkbox',
  defaultValue: false,
  label: 'Чат заблокирован',
  admin: {
    description: 'Если включено, пациент не может отправлять сообщения',
  },
}
```

**Логика:**
- По умолчанию `false` (чат доступен)
- После завершения консультации (`completed`) чат **остается открытым** - пациент может продолжать писать
- Врач может заблокировать чат кнопкой "Запретить пациенту писать" (устанавливает `chatBlocked = true`)
- Врач может разблокировать чат кнопкой "Разрешить пациенту писать" (устанавливает `chatBlocked = false`)
- Когда `chatBlocked = true`, пациент не может отправлять сообщения, но врач **всегда может**
- Пациент видит сообщение "Врач окончательно завершил консультацию" когда чат заблокирован

## Socket события для управления консультацией

### Клиентские события (отправка)

| Событие | Описание | Кто может отправлять |
|---------|----------|---------------------|
| `consultation-start` | Начало консультации | Только врач |
| `consultation-end` | Завершение консультации | Только врач |
| `chat-block` | Блокировка чата | Только врач (после завершения) |
| `chat-unblock` | Разблокировка чата | Только врач (после завершения) |

### Серверные события (получение)

| Событие | Описание | Данные |
|---------|----------|--------|
| `consultation-started` | Консультация началась | `{ appointmentId, message }` (message - системное сообщение) |
| `consultation-ended` | Консультация завершена | `{ appointmentId, message }` (message - системное сообщение) |
| `chat-blocked` | Чат заблокирован | `{ appointmentId }` |
| `chat-unblocked` | Чат разблокирован | `{ appointmentId }` |

## UI компоненты

### Для врача (`/lk-med/chat`)

**Баннеры:**
- **Зеленый** - "Консультация начнется через..." (до начала)
- **Синий** - "Консультация в процессе" (активна)
- **Серый** - "Консультация завершена" (завершена)

**Кнопки:**
- **"Начать консультацию"** - до начала (выбор видео/чат)
- **"Видеозвонок"** - во время консультации
- **"Завершить консультацию"** - во время консультации
- **"Запретить пациенту писать"** - после завершения (если чат не заблокирован)
- **"Разрешить пациенту писать"** - после завершения (если чат заблокирован)

### Для пациента (`/lk/chat`)

**Баннеры:**
- **Зеленый** - "Консультация начнется через..."
- **Синий** - "Консультация в процессе"
- **Серый** - "Консультация завершена"

**Поле ввода:**
- **Активно** - всегда, пока чат не заблокирован (в том числе после завершения консультации)
- **"Врач окончательно завершил консультацию"** - если чат заблокирован (врач нажал "Запретить пациенту писать")

## Серверные хэндлеры

### consultationHandler.ts

```typescript
// src/lib/socket/handlers/consultationHandler.ts

// Начало консультации
createConsultationStartHandler(io, payload)
// -> Проверяет права врача
// -> Обновляет status на 'in_progress'
// -> Эмитит 'consultation-started' в комнату

// Завершение консультации
createConsultationEndHandler(io, payload)
// -> Проверяет права врача
// -> Обновляет status на 'completed'
// -> Эмитит 'consultation-ended' в комнату

// Блокировка чата
createChatBlockHandler(io, payload)
// -> Проверяет права врача
// -> Проверяет что консультация завершена
// -> Устанавливает chatBlocked = true
// -> Эмитит 'chat-blocked' в комнату

// Разблокировка чата
createChatUnblockHandler(io, payload)
// -> Проверяет права врача
// -> Проверяет что консультация завершена
// -> Устанавливает chatBlocked = false
// -> Эмитит 'chat-unblocked' в комнату
```

## Системные сообщения (isSystemMessage)

В коллекции `Messages` есть поле `isSystemMessage`:

```typescript
{
  name: 'isSystemMessage',
  type: 'checkbox',
  defaultValue: false,
  label: 'Системное сообщение',
  admin: {
    description: 'Если включено, сообщение отображается как системное уведомление',
  },
}
```

**Логика:**
- Системные сообщения создаются автоматически при определённых событиях (например, смена типа связи)
- У системных сообщений `sender = null` (не привязаны к пользователю или врачу)
- В UI отображаются по центру с горизонтальными линиями по бокам (не как пузырёк сообщения)
- Используются для информирования обеих сторон о важных изменениях в консультации

**Когда создаются:**
- При начале консультации врачом → "Врач начал консультацию"
- При завершении консультации врачом → "Врач завершил консультацию"
- При изменении пациентом предпочтительного способа связи → "Пациент изменил предпочтительный способ связи на {тип}"

**Отображение в UI:**
- Системные сообщения отображаются по центру с горизонтальными линиями по бокам
- Сверху отображается дата/время сообщения (формат: "Сегодня, 14:30" или "25 апреля, 14:30")
- Компонент: `SystemMessageBubble` в `src/components/chat/message-bubble.tsx`

## Предпочтительный тип связи (connectionType)

В коллекции `Appointments` есть поле `connectionType`:

```typescript
{
  name: 'connectionType',
  type: 'select',
  required: false,
  defaultValue: 'chat',
  label: 'Вид связи',
  options: [
    { label: 'Чат', value: 'chat' },
    { label: 'Аудио', value: 'audio' },
    { label: 'Видео', value: 'video' },
  ],
  admin: {
    description: 'Предпочтительный способ связи пациента',
  },
}
```

**Логика:**
- Пациент выбирает предпочтительный способ связи при записи на консультацию
- По умолчанию `'chat'` (текстовый чат)
- Значение можно изменить после записи (пока консультация не завершена)
- Врач видит выбранный тип связи в хедере чата
- При изменении типа связи создаётся системное сообщение для информирования врача

### Где можно установить/изменить connectionType

| Место | Путь | Кто может изменять |
|-------|------|-------------------|
| Запись на консультацию | `/doctor/[id]/booking` | Пациент (при создании записи) |
| Чат пациента | `/lk/chat?appointment={id}` | Пациент (dropdown в хедере) |

### UI для пациента

В `ChatHeader` для пациента отображается dropdown:
- Иконка текущего типа связи + название + стрелка
- При клике — выпадающее меню с тремя опциями (Чат, Аудио, Видео)
- При выборе → отправляется socket событие `connection-type-change`
- Создаётся системное сообщение в чате

### UI для врача

Врач видит только текущий тип связи (без возможности изменения):
- Иконка + название типа связи в хедере чата

### Socket события

| Событие | Описание | Кто отправляет |
|---------|----------|----------------|
| `connection-type-change` | Изменение типа связи | Пациент |
| `connection-type-changed` | Уведомление об изменении | Сервер → всем в комнате |

**Серверный хэндлер:** `createConnectionTypeChangeHandler()` в `src/lib/socket/handlers/consultationHandler.ts`
