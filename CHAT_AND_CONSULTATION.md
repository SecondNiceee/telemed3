# Логика чата и консультации

## Обзор

Этот документ описывает полную логику работы чата и консультаций в телемедицинской платформе, включая управление статусами консультации и блокировку чата через WebSocket.

---

## Статусы консультации (Appointment)

| Статус | Описание |
|--------|----------|
| `confirmed` | Консультация подтверждена, ожидает начала |
| `in_progress` | Консультация активна (врач начал) |
| `completed` | Консультация завершена |
| `cancelled` | Консультация отменена |

---

## Поле chatBlocked

В коллекции `Appointments` есть поле `chatBlocked`:

```typescript
{
  name: 'chatBlocked',
  type: 'checkbox',
  defaultValue: false,
  label: 'Чат заблокирован',
  admin: {
    description: 'Если включено, пациент не может отправлять сообщения',
  },
}
```

**Логика:**
- По умолчанию `false` (чат доступен)
- После завершения консультации (`completed`) чат **остается открытым** - пациент может продолжать писать
- Врач может заблокировать чат кнопкой "Запретить пациенту писать" (устанавливает `chatBlocked = true`)
- Врач может разблокировать чат кнопкой "Разрешить пациенту писать" (устанавливает `chatBlocked = false`)
- Когда `chatBlocked = true`, пациент не может отправлять сообщения, но врач **всегда может**
- Пациент видит сообщение "Врач окончательно завершил консультацию" когда чат заблокирован

## Socket события для управления консультацией

### Клиентские события (отправка)

| Событие | Описание | Кто может отправлять |
|---------|----------|---------------------|
| `consultation-start` | Начало консультации | Только врач |
| `consultation-end` | Завершение консультации | Только врач |
| `chat-block` | Блокировка чата | Только врач (после завершения) |
| `chat-unblock` | Разблокировка чата | Только врач (после завершения) |

### Серверные события (получение)

| Событие | Описание | Данные |
|---------|----------|--------|
| `consultation-started` | Консультация началась | `{ appointmentId }` |
| `consultation-ended` | Консультация завершена | `{ appointmentId }` |
| `chat-blocked` | Чат заблокирован | `{ appointmentId }` |
| `chat-unblocked` | Чат разблокирован | `{ appointmentId }` |

## Поток управления консультацией

### 1. Начало консультации

```
Врач                          Сервер                        Пациент
  │                              │                              │
  │ 1. Нажимает "Начать          │                              │
  │    консультацию"             │                              │
  │                              │                              │
  │ emit('consultation-start')   │                              │
  │ ─────────────────────────────>                              │
  │                              │                              │
  │                              │ 2. Проверка прав врача       │
  │                              │    Обновление БД:            │
  │                              │    status -> 'in_progress'   │
  │                              │                              │
  │                              │ emit('consultation-started') │
  │ <────────────────────────────┼─────────────────────────────>│
  │                              │                              │
  │ 3. UI обновляется            │               4. UI пациента │
  │    - Баннер "В процессе"     │                  обновляется │
  │    - Кнопка "Завершить"      │                  в реальном  │
  │                              │                  времени     │
```

### 2. Завершение консультации

```
Врач                          Сервер                        Пациент
  │                              │                              │
  │ 1. Нажимает "Завершить       │                              │
  │    консультацию"             │                              │
  │                              │                              │
  │ emit('consultation-end')     │                              │
  │ ─────────────────────────────>                              │
  │                              │                              │
  │                              │ 2. Обновление БД:            │
  │                              │    status -> 'completed'     │
  │                              │                              │
  │                              │ emit('consultation-ended')   │
  │ <────────────────────────────┼─────────────────────────────>│
  │                              │                              │
  │ 3. UI обновляется            │               4. UI пациента │
  │    - Баннер "Завершена"      │                  показывает  │
  │    - Кнопка "Запретить       │                  "Завершена" │
  │      пациенту писать"        │                              │
```

### 3. Блокировка/Разблокировка чата

```
Врач                          Сервер                        Пациент
  │                              │                              │
  │ 1. Нажимает "Запретить       │                              │
  │    пациенту писать"          │                              │
  │    (доступно после           │                              │
  │    завершения)               │                              │

  │                              │                              │
  │                              │ 2. Проверка:                 │
  │                              │    - Консультация завершена? │
  │                              │    Обновление БД:            │
  │                              │    chatBlocked -> true       │
  │                              │                              │
  │                              │ emit('chat-blocked')         │
  │ <────────────────────────────┼─────────────────────────────>│
  │                              │                              │
  │ 3. UI: Кнопка меняется на    │               4. UI пациента │
  │    "Разрешить пациенту       │                  - Поле ввода│
  │    писать"                   │                    заменяется│
  │                              │                    сообщением│
  │                              │                  "Врач       │
  │                              │                  окончательно│
  │                              │                  завершил    │
  │                              │                  консультацию"│
  │                              │                              │
  │ 5. Нажимает "Разрешить       │                              │
  │    пациенту писать"          │                              │
  │                              │                              │
  │ emit('chat-unblock')         │                              │
  │ ─────────────────────────────>                              │
  │                              │                              │
  │                              │ 6. Обновление БД:            │
  │                              │    chatBlocked -> false      │
  │                              │                              │
  │                              │ emit('chat-unblocked')       │
  │ <────────────────────────────┼─────────────────────────────>│
  │                              │                              │
  │ 7. UI: Кнопка меняется на    │               8. UI пациента │
  │    "Запретить пациенту       │                  - Поле ввода│
  │    писать"                   │                    снова     │
  │                              │                    активно   │
```

## UI компоненты

### Для врача (`/lk-med/chat`)

**Баннеры:**
- **Зеленый** - "Консультация начнется через..." (до начала)
- **Синий** - "Консультация в процессе" (активна)
- **Серый** - "Консультация завершена" (завершена)

**Кнопки:**
- **"Начать консультацию"** - до начала (выбор видео/чат)
- **"Видеозвонок"** - во время консультации
- **"Завершить консультацию"** - во время консультации
- **"Запретить пациенту писать"** - после завершения (если чат не заблокирован)
- **"Разрешить пациенту писать"** - после завершения (если чат заблокирован)

### Для пациента (`/lk/chat`)

**Баннеры:**
- **Зеленый** - "Консультация начнется через..."
- **Синий** - "Консультация в процессе"
- **Серый** - "Консультация завершена"

**Поле ввода:**
- **Активно** - всегда, пока чат не заблокирован (в том числе после завершения консультации)
- **"Врач окончательно завершил консультацию"** - если чат заблокирован (врач нажал "Запретить пациенту писать")

## Серверные хэндлеры

### consultationHandler.ts

```typescript
// src/lib/socket/handlers/consultationHandler.ts

// Начало консультации
createConsultationStartHandler(io, payload)
// -> Проверяет права врача
// -> Обновляет status на 'in_progress'
// -> Эмитит 'consultation-started' в комнату

// Завершение консультации
createConsultationEndHandler(io, payload)
// -> Проверяет права врача
// -> Обновляет status на 'completed'
// -> Эмитит 'consultation-ended' в комнату

// Блокировка чата
createChatBlockHandler(io, payload)
// -> Проверяет права врача
// -> Проверяет что консультация завершена
// -> Устанавливает chatBlocked = true
// -> Эмитит 'chat-blocked' в комнату

// Разблокировка чата
createChatUnblockHandler(io, payload)
// -> Проверяет права врача
// -> Проверяет что консультация завершена
// -> Устанавливает chatBlocked = false
// -> Эмитит 'chat-unblocked' в комнату
```

## Права доступа

| Действие | Врач | Пациент |
|----------|------|---------|
| Начать консультацию | Да | Нет |
| Завершить консультацию | Да | Нет |
| Заблокировать чат | Да (после завершения) | Нет |
| Разблокировать чат | Да (после завершения) | Нет |
| Отправлять сообщения (до завершения) | Да | Да |
| Отправлять сообщения (после завершения, чат НЕ заблокирован) | Да | Да |
| Отправлять сообщения (чат заблокирован) | Да | Нет |
| Читать историю чата | Да | Да |
